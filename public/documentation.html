Book API documentation

API goal

Our book API is set to build the server-side component of a books web application. The web application will provide users with access to information about different books about business, management, personal growth, emotional intelligence, and many more. Users will be able to sign up, update their personal information, and create a list of their favorite books. It is necessary to login (and therefore, register) before doing absolutely anything. To register, go to /register and add name, password, email, and a birth date (format yyyy/mm/dd).

URL Endpoints

  • GET "/API/books": Deploys as a response a list of all the books in our Database. Requires registration and login (see POST login and POST register).
  • GET "/API/books/:title" : Deploys as a response the selected books by title. Requires registration and login (see POST login and POST register).
  • POST "/users" : allows to sign up. Requires a name, password, email, and a birth date (format yyyy/mm/dd). Responses with the brand-new created user in JSON.
  • POST "/users/:Username/Books/:BookID" : allows you to add a book to your favourite books list. Requires registration and login (see POST login and POST register)
  • POST "/login" : allows you to access to your account by inserting user name and password. Afterwards, you will receive a JWT with which you will be allowed to enter to the rest of the endpoints.
  • PUT "/users/:Name" : updates user's info. Expects JSON in this format: { Name: String, (required) Password: String, (required) Email: String, (required) Birthday: Date }. Requires registration and login (see POST login and POST register)
  • DELETE "/users/:Username/Books/:BookID" : allows you to remove a book to your favourite books list. Requires registration and login (see POST login and POST register)
  • DELETE "/users/:Username" : allows you to remove your user. Requires registration and login (see POST login and POST register)
